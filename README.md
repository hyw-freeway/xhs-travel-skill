# xhs-travel-guide：基于小红书的自动旅游攻略生成器

**xhs-travel-guide** 是一个基于小红书内容的「自动化旅游攻略生成」 Skill，属于本项目的小红书自动化能力集合之一。  
它通过接入 [xiaohongshu-mcp](https://github.com/xpzouying/xiaohongshu-mcp) 服务，自动搜索小红书上的高质量攻略笔记，聚合多篇信息，生成一份结构化、可直接发小红书的 Markdown 旅游攻略（含行程、避坑、交通、门票和溯源链接）。

适配 **Agent Skills 开放标准**，可在 **Claude Code / Cursor / OpenClaw / 其他支持 Agent Skills 的平台** 中使用。

---

## 功能特点

- **一键生成完整旅游攻略**
  - 输入「目的地 + 天数」，例如：`/xhs-travel-guide 深圳 3天`
  - 自动输出 `{目的地}-{天数}游攻略-小红书综合.md`，结构固定、可直接复制到小红书笔记中使用

- **基于真实小红书内容**
  - 多维度搜索小红书攻略：主干行程、避坑/雷点、交通、门票/预约；四维度合并后按 feed_id 去重，最多 12 篇拉详情
  - 只选 **高互动、高质量** 的公开笔记（优先标题含「保姆级」「血泪总结」「亲测」等），并综合**高赞评论**中的补充信息

- **每条建议都有溯源链接**
  - 每一天行程至少关联 1 条与**当日内容匹配**的小红书笔记；匹配的多条可都贴上
  - 「当日参考攻略」挂前会校验笔记是否与该日景点/区域/主题相关，不相关的不挂
  - 交通、避坑、门票模块都附有对应的笔记链接
  - 链接统一为：`https://www.xiaohongshu.com/explore/{feed_id}`（不带隐私参数）

- **为小红书笔记量身定制的结构**
  - **🌟 行程总览表**：日期、区域、主要安排、路线规划及推荐理由
  - **✈️ 交通与住宿指南**：大交通、市内交通、住宿区域（附参考笔记链接）
  - **📅 每日详细行程**：上午/下午/晚上安排 + 「当日参考攻略」链接
  - **🎫 门票预约与避坑指南**：需要预约的景点 + 防坑要点（每条有对应小红书来源）
  - **🔗 参考小红书笔记汇总表**：标题 / 作者 / 链接 / 核心贡献，一目了然

- **读写安全**
  - 只调用 **搜索与内容读取** 的 MCP 工具（`search_feeds`、`get_feed_detail` 等）
  - 不做点赞、评论、发笔记等写操作
  - 自动在文末标注「仅供参考」，避免把网络攻略当官方权威

## 依赖与整体架构

本 Skill 基于以下组件：

- **xiaohongshu-mcp**（必需）  
  使用 Go 实现的小红书 MCP 服务，封装了登录、搜索、笔记详情、用户主页等 API。  
  仓库地址：[xiaohongshu-mcp](https://github.com/xpzouying/xiaohongshu-mcp)

- **Agent Skills 集合**（本项目）  
  目录结构示例：
  - `.claude/skills/`（或 `skills/`）
    - `SKILL.md`：整个技能集合的入口
    - `xhs-login/`：登录管理（扫码登录、状态检查、重新登录）
    - `xhs-search/`：搜索笔记
    - `xhs-explore/`：推荐流与笔记详情
    - `xhs-interact/`：点赞/收藏/评论/回复
    - `xhs-profile/`：用户主页
    - `post-to-xhs/`：发布图文/视频笔记
    - `setup-xhs-mcp/`：一键安装/配置 MCP 服务
    - **`xhs-travel-guide/`**：本 README 对应的自动生成旅游攻略 Skill

## 安装

### 1. 安装并运行 xiaohongshu-mcp

请参考上游项目的官方文档，这里只给出一个典型示意流程：

```bash
git clone https://github.com/xpzouying/xiaohongshu-mcp.git
cd xiaohongshu-mcp

# 安装依赖
go mod tidy

# 运行服务（实际命令以仓库文档为准）
go run ./main.go
```

也可以使用 Docker / docker-compose，详见 [xiaohongshu-mcp/docker](https://github.com/xpzouying/xiaohongshu-mcp/tree/main/docker)。

确保 MCP 服务启动后，可以在本地通过 HTTP 或 MCP 协议访问。

### 2. 安装 Agent Skills（以 Claude Code / Cursor 为例）

将本项目的 skills 目录复制到你的环境中：

```bash
# 项目级：将本仓库的 .claude/skills 复制到你的项目根目录下
cp -r .claude/skills/ /path/to/your/project/.claude/skills/

# 或全局级（以 Mac/Linux 为例）
cp -r .claude/skills/ ~/.claude/skills/
```

此时，`xhs-travel-guide` 将作为一个可用 Skill 被客户端发现。

### 3. 在编辑器中加载 Skills

不同客户端加载方式略有差异，一般为：

- 保证 `.claude/skills/` 已存在并包含 `xhs-travel-guide/` 子目录
- 按客户端要求配置 MCP 连接（如 `.cursor/mcp.json` 指向 xiaohongshu-mcp 服务）
- 重启会话 / 重启编辑器
- 在对话中输入 `/xhs-travel-guide` 测试 Skill 是否可用

## 使用方法

### 基础命令

```text
# 1. 首次使用：安装 MCP 服务和登录
/setup-xhs-mcp      # 引导安装部署 xiaohongshu-mcp
/xhs-login          # 扫码登录小红书账号

# 2. 生成旅游攻略（综合小红书攻略帖，带溯源链接）

# 单一目的地 + 按天数
/xhs-travel-guide 深圳 3天
/xhs-travel-guide 西双版纳 7天

# 多目的地联游
/xhs-travel-guide 西双版纳+普洱+景迈山 7天

# 可附加偏好（视宿主模型/Agent 是否支持自然语言解析）
# 例如：带父母、不自驾、想少爬山等
/xhs-travel-guide 深圳 3天 带父母 不自驾
```

### 输入约定（来自 xhs-travel-guide/SKILL.md）

- **必填**
  - **目的地**：如「西双版纳」「西双版纳+普洱+景迈山」，支持多城市/区域组合
  - **天数或日期范围**：如「7 天」「4.29–5.7」

- **可选**
  - **偏好**：如「无自驾」「带老人小孩」「穷游」「深度游」

当缺少关键信息（目的地/天数）时，Skill 会先向用户补问，再继续执行。

## 工作原理

### 1. 需求确认与画像构建

从自然语言对话中提取：

- 目的地（单城市/多城市）
- 天数或出行日期
- 出行人群（亲子/情侣/和父母/朋友团）
- 交通偏好（自驾/无自驾/公共交通为主）
- 预算偏好（穷游/常规/小资/奢华）

### 2. 多维度搜索小红书（search_feeds）

按以下矩阵对小红书发起搜索：

| 维度       | 搜索关键词示例                         | 排序方式   | 目的                         |
|------------|----------------------------------------|------------|------------------------------|
| 主干行程   | `{目的地} {天数}日游 攻略`            | 综合       | 构建整体行程骨架             |
| 避坑/雷点  | `{目的地} 旅游 避坑` / `劝退`         | 最多收藏   | 收集真实负面反馈和注意事项   |
| 交通出行   | `{目的地} 交通 攻略` / 机场/高铁到市区 | 综合       | 明确市内及城际交通方式       |
| 门票/预约  | `{目的地} 景点 门票 预约`            | 最新       | 获取最新预约渠道和抢票提示   |

每个维度选取前 5–10 条互动较高的笔记，记录其 `feed_id` 与 `xsec_token`；四维度合并后按 feed_id 去重，总候选不超过 18 条。

### 3. 深度内容提取（get_feed_detail）

合并去重后按互动量与标题关键词排序，**最多**对 12 篇笔记调用 `get_feed_detail`，并抽取：

- 具体景点名称、游玩时长、门票价格
- 餐厅名称及招牌菜
- 真实避坑点（如「排队 3 小时不值得」「强制消费」「环境太差」等）
- 交通信息（推荐地铁/公交/自驾/打车路线）

**评论**：默认仅用首屏约 10 条评论；仅对 2–3 篇核心笔记可设 `load_all_comments=true` 且 `limit=15`，综合高赞评论中的补充信息。  
同时建立「信息 ↔ 来源笔记」的映射，并标注每篇笔记覆盖的行程日/主题，用于后续挂「当日参考攻略」时只贴内容相关的链接。

### 4. 合成结构化 Markdown

基于抽取的信息（笔记正文 + 高赞评论）生成攻略。**写每日行程时**先根据映射筛选出与该日景点/区域/主题真正相关的笔记，再挂到「当日参考攻略」——不匹配的不挂在该日，可放入文末汇总表；匹配的多条可都贴。

要求：按时间线组织（Day 1 / Day 2 / …）；每一天至少 1 条与当日内容匹配的链接；交通、避坑、门票模块各至少 1 条来源链接；链接统一为 `https://www.xiaohongshu.com/explore/{feed_id}`。

输出结构固定包括：🌟 行程总览表、✈️ 交通与住宿指南、📅 每日详细行程（含「💡 当日参考攻略」）、🎫 门票预约与避坑指南、🔗 参考小红书笔记汇总表。

### 5. 输出结果文件

- 默认文件名示例：`西双版纳-普洱-景迈山七日游攻略-小红书综合.md`、`深圳三日游攻略-小红书综合.md`
- 可直接：
  - 粘贴到小红书后台发布
  - 少量手动润色、替换为自己的实拍照片后发文
  - 作为团队内部出行方案的基础模板

## 适用场景

- **旅游博主 / 小红书创作者**  
  想快速「扫一遍」某地高质量攻略，并整理成一篇结构清晰、有真实来源的保姆级笔记。

- **旅游从业者 / 旅行社**  
  制作目的地产品方案时，用于快速拉取行程/避坑/交通/门票的「用户视角」素材。

- **普通用户**  
  计划自由行，不想自己一篇一篇看完几十条笔记，希望 AI 帮你综合出一份「抄作业」底稿。

## 注意事项

- **信息来源**  
  所有内容均来自 **小红书公开笔记**，Skill 只是做聚合、提炼与排版，不代表任何官方机构或景区的立场。

- **时效性**  
  景点开放时间、预约规则、票价、交通方式等可能随时间变化，生成时会优先查阅「半年内」或「较新的」笔记，但仍建议你：
  - 出行前务必对关键信息（如门票预约、通行证要求、自驾限行）做一次官方确认
  - 将文档中的「仅供参考」提示保留

- **隐私与账号安全**
  - Skill 会通过 `xhs-login` 引导扫码登录，但不会读取或泄露你的账号密码
  - `xhs-travel-guide` 不会代表你发文、点赞、评论或操作粉丝，只做只读聚合
